# 部署检查清单

## ✅ 已完成的配置

1. ✅ Base URL 配置正确（生产环境使用 `/sweekli-website/`）
2. ✅ GitHub Actions 工作流已配置
3. ✅ 所有图片文件已提交到 Git
4. ✅ 构建输出正确（dist 文件夹包含所有文件）

## 🔧 需要手动检查的步骤

### 1. 确认 GitHub Pages 设置

1. 进入仓库：https://github.com/Pop-Up-Limited/sweekli-website
2. 点击 **Settings** > **Pages**
3. 确认：
   - **Source**: 选择 **GitHub Actions**（不是 branch）
   - 如果显示 "Your site is live at..." 说明已启用

### 2. 检查 GitHub Actions

1. 进入仓库，点击 **Actions** 标签
2. 查看最新的工作流：
   - 应该显示 "Deploy to GitHub Pages"
   - 状态应该是绿色 ✓（成功）
   - 如果失败，点击查看错误信息

### 3. 等待部署完成

- 首次部署需要 2-5 分钟
- 部署完成后，网站地址：https://pop-up-limited.github.io/sweekli-website/

### 4. 如果还是 404

1. **清除浏览器缓存**（Ctrl/Cmd + Shift + R）
2. **等待几分钟**后刷新
3. **检查 URL**：确保访问的是 `https://pop-up-limited.github.io/sweekli-website/`（注意末尾的 `/`）

## 🚨 常见问题

### 问题：还是显示 404 或加载 `/src/main.ts`

**原因**：GitHub Pages 可能还在使用旧的部署

**解决方法**：
1. 进入 **Actions** 标签
2. 点击最新的工作流
3. 点击 **Re-run all jobs** 重新部署
4. 等待部署完成

### 问题：图片不显示

**原因**：图片路径问题

**解决方法**：
1. 检查浏览器控制台，查看具体哪些图片 404
2. 确认图片路径是否正确（应该是 `/sweekli-website/images/...`）
3. 如果路径错误，需要修复代码中的图片路径
